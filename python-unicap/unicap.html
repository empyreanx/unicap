enumerate_properties(...)
Returns a list of all properties supported by the device.
 
Each list entry is a dictionary with at least the following fields:
name: A unique string specifying the property
category: A string describing the property category
unit: A string describing the property unit
 
The property may have one of the following keys:
value: A float value
menu_item: A string value
 
The property may have the following optional keys:
range: A (min,max) tuple specifying the range in which the 'value' may be
values: A list of float values specifying the allowed values for the 'value' key
menu: A list of strings specifying the allowed values for the 'menu_item' key
flags: A list of strings specifying flags that are set on the property
capabilities: A list of string specifying the allowed flags for the property
 
Possible flags are:
'manual': Indicates manual operation of the property
'auto': Indicates automatic setting of the property
'one push': Indicates a one-time automatic action of the property
'read only': Indicates that the property may only be read
'write only': Indicates that the values read with get_property do not reflect the actual setting

wrap_gpointer(ptr)
Creates a new ImageBuffer instance from a gpointer.
The gpointer should point to a C struct of (unicap_data_buffer_t*) type. 
This function is used to create an ImageBuffer instance from the gpointer that is passed to the signal handlers eg. of a unicapgtk.VideoDisplay widget
The image data is not copied and no data buffer memory is allocated by this method so this ImageBuffer may only be used as long as 
the underlying unicap_data_buffer is valid ( ie. only inside the signal handler function ).
